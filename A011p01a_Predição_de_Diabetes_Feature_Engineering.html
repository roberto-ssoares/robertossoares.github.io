Sobre o conjunto de dados

Contexto

  • Este conjunto de dados é originalmente do National Institute of Diabetes and Digestive and Kidney Diseases.
  • O objetivo do conjunto de dados é prever diagnosticamente se um paciente tem ou não diabetes, com base em certas medições diagnósticas incluídas no conjunto de dados.
  • Várias restrições foram colocadas na seleção dessas instâncias de um banco de dados maior.
  • Em particular, todos os pacientes aqui são mulheres com pelo menos 21 anos de idade e de ascendência indígena Pima.

Conteúdo

  • Os conjuntos de dados consistem em várias variáveis ​​preditoras médicas e uma variável alvo, Outcome.
  • As variáveis ​​preditoras incluem o número de gestações que a paciente teve, seu IMC, nível de insulina, idade e assim por diante.

Agradecimentos

  • Smith, JW, Everhart, JE, Dickson, WC, Knowler, WC, & Johannes, RS (1988). Usando o algoritmo de aprendizagem ADAP para prever o início do diabetes mellitus .
  • Em Anais do Simpósio sobre Aplicações de Computador e Assistência Médica (pp. 261--265). IEEE Computer Society Press.

Função de Metadados:

Você pode gerar um DataFrame de metadados utilizando Spark SQL. Abaixo está a função equivalente que gera o mesmo resultado usando SQL em PySpark:

#from pyspark.sql.functions import expr

def gerar_metadados_sql(table_name: str):

    # Obtendo as colunas e seus tipos
    dataframe = spark.table(table_name)
    colunas = dataframe.columns
    tipos = [str(dataframe.schema[col].dataType) for col in colunas]

    # Lista para armazenar o resultado de cada coluna
    metadados_list = []

    # Iterando sobre cada coluna para gerar os metadados
    for col, dtype in zip(colunas, tipos):
        sql_query = f"""
        SELECT
            '{table_name}' AS nome_tabela,
            '{col}' AS nome_variavel,
            '{dtype}' AS tipo,
            COUNT(*) AS total_linhas,
            COUNT(CASE WHEN `{col}` IS NULL THEN 1 END) AS qt_nulos,
            (COUNT(CASE WHEN `{col}` IS NULL THEN 1 END) / COUNT(*)) * 100 AS percent_nulos,
            ROUND(AVG(`{col}`),2) AS avg_nulos,
            COUNT(DISTINCT `{col}`) AS cardinalidade

        FROM
            {table_name}
        """

        # Executando a consulta SQL para cada coluna e coletando o resultado
        metadados_df = spark.sql(sql_query)
        metadados_list.append(metadados_df)

    # Unindo os resultados de todas as colunas
    final_metadados_df = metadados_list[0]
    for df in metadados_list[1:]:
        final_metadados_df = final_metadados_df.union(df)

    return final_metadados_df

            COUNT(CASE WHEN `{col}` = 0 THEN 1 END) AS qt_zeros,
            ROUND((COUNT(CASE WHEN `{col}` = 0 THEN 1 END) / COUNT(*)) * 100,2) AS percent_zeros,
            ROUND(AVG(`{col}`),2) AS avg_zeros,
            COUNT(DISTINCT `{col}`) AS cardinalidade
        FROM
            {table_name}
        """

        # Executando a consulta SQL para cada coluna e coletando o resultado
        metadados_df = spark.sql(sql_query)
        metadados_list.append(metadados_df)

    # Unindo os resultados de todas as colunas
    final_metadados_df = metadados_list[0]
    for df in metadados_list[1:]:
        final_metadados_df = final_metadados_df.union(df)

    return final_metadados_df

Observações:

Em um dataset de saúde, como esse de diabetes, substituir valores zero pela média da variável pode não ser a abordagem mais adequada em todos os casos, pois o zero em variáveis como "Glucose", "BloodPressure", "SkinThickness", "Insulin", e "BMI" não é um valor válido do ponto de vista clínico. Zero pode indicar valores faltantes ou erros de entrada de dados.

Aqui estão algumas considerações e práticas recomendadas para lidar com esses casos:

  1. Entender o Significado do Zero
  • Para variáveis como "BloodPressure" e "Glucose", valores zero são clinicamente irreais (um ser humano não pode ter pressão arterial ou glicose igual a zero). Portanto, zeros devem ser tratados como valores ausentes ou inválidos.
  1. Substituição pela Média
  • A substituição de valores faltantes pela média pode ser uma técnica comum, mas em variáveis de saúde, isso pode suavizar os dados de forma que padrões importantes sejam perdidos. Por exemplo, substituir zeros na "Glucose" pela média pode distorcer a realidade do dataset e fazer com que você perca variabilidade importante.
  1. Imputação com Métodos Mais Avançados
  • Métodos de imputação mais avançados podem ser utilizados para preservar melhor as características dos dados:
    • Imputação pela Mediana: Para variáveis de saúde, a mediana pode ser uma escolha melhor do que a média, especialmente se os dados tiverem outliers.
    • Imputação com KNN (K-Nearest Neighbors): Considera os valores das variáveis de registros similares para imputar valores ausentes.
    • Modelos de Regressão: Para algumas variáveis, pode-se treinar um modelo de regressão que preveja os valores ausentes com base em outras variáveis do dataset.
    • Multivariate Imputation by Chained Equations (MICE): Imputação múltipla levando em conta as inter-relações entre as variáveis.
  1. Considerar Exclusão dos Valores Faltantes
  • Dependendo da proporção de zeros em cada variável:
    • Exclusão de Linhas: Se o número de registros com zero for pequeno, pode-se considerar remover essas linhas.
    • Imputação Selecionada: Algumas variáveis com poucos zeros, como "Glucose" (0.65% de valores nulos), podem ser imputadas de forma relativamente simples, mas variáveis com muitos zeros, como "Outcome" (65.1% de valores faltantes), podem precisar de uma análise mais cuidadosa.
  1. Análise Exploratória Adicional
  • Antes de tomar decisões sobre o tratamento dos zeros, você pode:
    • Verificar a distribuição das variáveis com histograma ou boxplot.
    • Testar as correlações entre variáveis para entender se há relações fortes entre os valores válidos de outras variáveis e os zeros.

Conclusão

A imputação de zeros em variáveis de saúde deve ser feita com cuidado, e o uso de métodos mais avançados, como mediana, KNN ou modelos de regressão, é recomendado para manter a integridade dos dados.

Apenas substituir pelos valores médios pode não refletir corretamente os padrões de saúde dos pacientes.

Imputar os valores zero nas variáveis de saúde com a mediana:

#Para imputar os valores zero nas variáveis de saúde com a mediana:
#from pyspark.sql.functions import col, when, avg

# Substituir zeros pela mediana da variável
def substituir_zeros(df, col_name):
    median_value = df.approxQuantile(col_name, [0.5], 0.0)[0]
    df = df.withColumn(col_name, when(col(col_name) == 0, median_value).otherwise(col(col_name)))
    return df

# Aplicar nas variáveis
variaveis_saude = ['Glucose', 'BloodPressure', 'SkinThickness', 'Insulin', 'BMI']
for var in variaveis_saude:
    df_diabetes = substituir_zeros(df_diabetes, var)

Observações:

  • Essa abordagem vai imputar os zeros nas variáveis de saúde usando a mediana, o que é mais robusto em relação à média.
